# MRU Calibration Analysis Project

## Project Overview
This is a Python-based data analysis and reporting project used for the calibration and testing of a Motion Reference Unit (MRU). The project analyzes temporal test data and generates a formal PDF Calibration Certificate based on the calculated metrics. 

Main technologies and libraries used:
- **Python**: Primary programming language.
- **Jupyter Notebooks**: Used for data processing and metric calculation (`Analysis.ipynb`).
- **Pandas, NumPy, SciPy**: Used for data manipulation, filtering, signal processing, and temporal model regression.
- **Matplotlib**: Used for generating accuracy plots and error graphs.
- **ReportLab**: Used for rendering the final PDF certificate.

## Directory Structure Overview
- `Analysis.ipynb`: Main notebook for analyzing the MRU data.
- `info.json`: Metadata about the test suite setup.
- `Format/`, `Reference/`, `Target/`: Directories containing specific `.csv` datasets and corresponding sub-notebooks (`Analysis.ipynb`) for distinct calibration analysis steps.
- `Abkp/`: Contains the `TemporalFit.py` script which applies `scipy.optimize.curve_fit` for temporal model regression and performance validation metrics.
- `Certificate/`: Contains the logic to generate the final PDF certificate.
  - `index.py`: The entry point script that reads the metrics and plots to generate the PDF.
  - `images/`: Contains visual assets like logos and signatures.
  - `results/`: Contains the calculated metrics (`metrics.json`) and plots (`error.png`, `ref_vs_model.png`) for each axis (roll, pitch, yaw, wx, etc.) generated by the analysis phase.
  - `release/`: The output directory for the generated `certificate.pdf`.

## Building and Running

1. **Dependencies**: 
   Ensure the required Python libraries are installed. 
   ```bash
   pip install pandas numpy scipy matplotlib reportlab jupyter
   ```

2. **Data Analysis**: 
   Run the `Analysis.ipynb` Jupyter Notebooks (at the root and inside the specific test subdirectories like `Format/`) to process the raw `.csv` data. This step generates the necessary `metrics.json` and plot images (`.png`) in the `Certificate/results/` directory.
   ```bash
   jupyter notebook Analysis.ipynb
   ```

3. **Certificate Generation**:
   Once the analysis is complete and the results are populated, run the `index.py` script to generate the final PDF certificate.
   ```bash
   cd Certificate
   python index.py
   ```
   The generated certificate will be saved to `Certificate/release/certificate.pdf`.

## Development Conventions
- **Object-Oriented Programming (OOP)**: The codebase utilizes OOP principles, as seen in the `ReportData` and calibration report classes in `Certificate/index.py`, as well as `TemporalFit` in `Abkp/TemporalFit.py`. Future additions should follow this class-based structure.
- **Clean Code & Indentation**: Maintain clean, properly indented code. Avoid unnecessary comments, relying instead on self-documenting variable and method names.
- **Data Handling**: Use `pandas` DataFrames for structured handling and processing of CSV datasets, and `numpy` for efficient array operations.
- **File Pathing**: Scripts rely on robust path resolution to resolve relative file paths properly and ensure execution from different working directories.
