# MRU Calibration Analysis Project

## Project Overview
This is a Python-based data analysis and reporting project used for the calibration and testing of a Motion Reference Unit (MRU). The project analyzes static and dynamic test data and generates a formal PDF Calibration Certificate based on the calculated metrics. 

Main technologies and libraries used:
- **Python**: Primary programming language.
- **Jupyter Notebooks**: Used for data processing and metric calculation (`Analysis.ipynb`).
- **Pandas, NumPy, SciPy**: Used for data manipulation, filtering, and numerical analysis.
- **Matplotlib**: Used for generating accuracy plots and error graphs.
- **ReportLab**: Used for rendering the final PDF certificate.

## Directory Structure Overview
- `Analysis.ipynb`: Main notebook for analyzing the MRU data.
- `info.json`: Metadata about the test suite (e.g., dynamic and static test counts).
- `Analysis/`: Contains sub-directories `DynamicTests/` and `StaticTests/` with their respective data files (`.csv`), reference/target data, and specific analysis notebooks.
- `Certificate/`: Contains the logic to generate the final PDF certificate.
  - `index.py`: The entry point script that reads the metrics and plots to generate the PDF.
  - `images/`: Contains visual assets like logos and signatures.
  - `results/`: Contains the calculated metrics (`metrics.json`) and plots (`error.png`, `ref_vs_model.png`) for each axis (roll, pitch, yaw, etc.) generated by the analysis phase.
  - `release/`: The output directory for the generated `certificate.pdf`.

## Building and Running

1. **Dependencies**: 
   Ensure the required Python libraries are installed. 
   ```bash
   pip install pandas numpy scipy matplotlib reportlab jupyter
   ```

2. **Data Analysis**: 
   Run the `Analysis.ipynb` Jupyter Notebooks (at the root and inside the `Analysis/` subdirectories) to process the raw `.csv` data. This step generates the necessary `metrics.json` and plot images (`.png`) in the `Certificate/results/` directory.
   ```bash
   jupyter notebook Analysis.ipynb
   ```

3. **Certificate Generation**:
   Once the analysis is complete and the results are populated, run the `index.py` script to generate the final PDF certificate.
   ```bash
   cd Certificate
   python index.py
   ```
   The generated certificate will be saved to `Certificate/release/certificate.pdf`.

## Development Conventions
- **Object-Oriented Programming (OOP)**: The codebase utilizes OOP principles, as seen in the `CalibrationReport` class in `Certificate/index.py`. Future additions should follow this class-based structure.
- **Clean Code & Indentation**: Maintain clean, properly indented code. Avoid unnecessary comments, relying instead on self-documenting variable and method names (e.g., `_drawInfoSection`, `_buildAccuracyRows`).
- **Data Handling**: Use `pandas` DataFrames for structured handling and processing of CSV datasets.
- **File Pathing**: Scripts rely on robust path resolution (e.g., using `os.path.dirname(os.path.abspath(__file__))`) to resolve relative file paths properly and ensure execution from different working directories.
